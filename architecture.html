Overview

Torus is a self-improving quality framework for Claude Code. It wraps every tool call with gate enforcement (PreToolUse blocking), tracks outcomes via a mentor system (PostToolUse), persists knowledge via LanceDB vector database, and orchestrates multi-agent work via teams and external scripts. The framework runs entirely through Claude Code's hook system — no modifications to Claude Code itself.

01
Mechanical Enforcement
sys.exit(2) blocks tool calls mechanically — not just behavioral instructions
02
Fail-Closed / Fail-Open
Tier 1 crash = block (safety). Tier 2+ crash = warn + continue (quality)
03
Session Isolation
Per-session state files keyed by session_id. Agents can't bleed state
04
Memory-First Workflow
Every fix/decision/discovery persisted to LanceDB. Compounding knowledge corpus
05
Causal Fix Tracking
Fingerprinted errors, strategy bans (3+ failures), outcome chains
06
Hot I/O on Ramdisk
~544 MB/s tmpfs vs ~1.2 MB/s disk. Async mirror for persistence

Gate Dispatch

  • Gates run in priority order from shared/gate_registry.GATE_MODULES
  • Q-learning optimization reorders Tier 2+ by historical block probability
  • Tool-scoped dispatch reduces unnecessary evaluations
  • Result caching for non-blocking gates (60s TTL)
  • Always-allowed tools bypass all gates: Read, Glob, Grep, WebFetch, WebSearch, AskUserQuestion, EnterPlanMode, ExitPlanMode, TaskCreate/Update/List/Get, all mcp__memory__* and mcp__analytics__* tools

MCP Servers

Memory Server — memory_server.py

Embedding: nomic-embed-text-v2-moe (768-dim). Storage: ~/data/memory/lancedb/ (LanceDB vector DB). Tag index: ~/data/memory/tags.db (SQLite, <1MB). ChromaDB: backup only at ~/data/memory/chroma.sqlite3.

3-tier memory classification: Tier 1 high-value memories boosted in ranking, Tier 2 standard memories at neutral weight, Tier 3 low-value memories penalized in ranking.
