Gate Pipeline

torus-framework — all 17 gates in execution order — 2026-03-06

Every tool call flows through enforcer.py which routes to gate_router.py. Gates are grouped into three tiers. Tier 1 is fail-closed — any exception blocks. Tiers 2 and 3 are fail-open. Always-allowed tools bypass the pipeline entirely.

GateResult Structure

field type description
blockedboolTrue halts the tool call (exit 2). False proceeds regardless of severity.
messagestrHuman-readable explanation shown to Claude in stderr when blocked.
gate_namestrIdentifier of the gate that produced this result (e.g. "read_before_edit").
severityinfo | warn | error | criticalDescriptive level. Does not control blocking — only blocked=True blocks.
escalationblock | ask | warn | allowInferred from blocked if omitted. "ask" triggers interactive confirmation.
duration_msfloatGate execution time in milliseconds. Used for performance monitoring.
metadatadictArbitrary additional data for audit log and downstream gates.

Dispatch Details

Q-Learning Reorder
Gate execution order adapts at runtime using Q-learning. Gates that block more often get promoted higher in the dispatch queue to fail fast.
Tool-Scoped Dispatch
Each gate declares which tool types it cares about. gate_router.py only dispatches gates relevant to the current tool call, reducing unnecessary work.
Result Caching
Gate results are cached with a 60-second TTL keyed by (gate, tool, input hash). Identical repeated calls skip recomputation.
Always-Allowed Bypass
Read, Glob, Grep, and other read-only tools are in the always-allowed list. They skip the gate pipeline entirely for zero overhead on safe operations.
Tier contract: Tier 1 gates are fail-closed — any Python exception within the gate code is treated as a block (enforcer.py catches and returns blocked=True). This ensures safety gates can never silently pass due to a bug. Tiers 2 and 3 are fail-open — exceptions produce a warning result and execution continues. The tier assignment is set in enforcer.py:TIER1_SAFETY_GATES.
