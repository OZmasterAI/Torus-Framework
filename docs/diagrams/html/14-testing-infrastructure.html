01 — test_framework.py — Orchestrator
test_framework.py 53 lines
Main test orchestrator. Discovers and runs all test modules in hooks/tests/, aggregates pass/fail counts, and exits with a non-zero code on any failure. Serves as the canonical entry point for CI and manual test runs — a single command to run all 11 focused test files plus shared module coverage.

04 — hooks/benchmarks/
benchmark_gates.py 458 lines
Gate latency benchmarks. Measures p50/p95/p99 latencies for each gate under synthetic load. Detects performance regressions vs. stored baselines in memory. Produces a JSON report with per-gate timing distributions.
benchmark_io.py 162 lines
I/O performance benchmarks. Measures ramdisk read/write throughput, state file serialization time, and audit log append latency. Ensures framework overhead stays within acceptable bounds under concurrent load.
Baseline tracking: Results saved with type:benchmark tags in memory for longitudinal regression detection across sessions.
05 — hooks/fuzz_gates.py
fuzz_gates.py 562 lines
Adversarial gate fuzzer. Generates malformed, edge-case, and boundary inputs for all gates — empty strings, unicode edge cases, deeply nested JSON, oversized payloads, and injection attempts. Verifies that gates fail gracefully and never raise unhandled exceptions that would cause exit code 1 instead of 2.

06 — shared/ Testing Tools
test_generator.py 533 lines
Auto-generates test scaffolding for new gates and shared modules. Reads the target file, extracts function signatures, and emits a pytest file with parametrized stubs. Reduces the overhead of adding coverage for new components.
mutation_tester.py 811 lines
Mutation testing engine. Applies systematic mutations (boolean negations, boundary off-by-ones, operator flips) to gate and shared module source code, then runs the test suite to verify tests catch the mutations. Identifies undertested code paths where tests pass despite logic errors.
verify_memory_maintenance.py 193 lines
Memory health verifier. Checks LanceDB table integrity, validates memory MCP server connectivity, scans for orphaned records, and verifies the sideband protocol file (hooks/.memory_last_queried) is being updated correctly.
07 — Test Categories
Safety
Tier-1 gate tests. Fail-closed: any exception during a safety gate check must produce exit code 2 (block), never exit code 1 (allow). Covers gate_01.py through gate_03.py — destructive commands, path traversal, force-push.
Quality
Gate tests for code quality enforcement: read-before-edit requirement, edit streak warnings, test file enforcement, and memory query verification (Gate 4 sideband).
Operational
Tests for session-management and diagnostic gates. Covers session startup/teardown, status reporting, health monitor integration, and analytics event recording.
Integration
End-to-end tests that simulate real tool call sequences: a full Bash command flowing through the complete gate chain, state accumulating across calls, and the final exit code verified. Catches cross-gate interaction bugs that unit tests miss.
Shared Core
Unit tests for the shared/ infrastructure layer. Covers state.py, gate_result.py, gate_router.py, circuit_breaker.py, ramdisk.py, memory_socket.py, and gate_timing.py in isolation.
Rule: All gates must have coverage in at least one location — either test_framework.py or a focused file in hooks/tests/.
