Mode Profiles
01
coding
default development mode — coding.md

"Smallest diff wins. Test first. Read before write."

Core discipline
  • Smallest diff wins — change only what is needed; no drive-by refactors
  • Test first — identify failing test before writing the fix
  • Read before write — understand callers, tests, types fully first
Code rules
  • Catch specific exceptions; never bare except:
  • Never swallow errors silently — log, re-raise, or handle explicitly
  • Type hints on all public function signatures
  • Remove unused imports; no # TODO placeholders
Communication
  • Skip preamble — go straight to implementation
  • When fixing: reproduce, fix, prove — in that order
02
debug
diagnosis and root-cause analysis — debug.md

"Reproduce first. One variable at a time. Trace, don't guess."

Core discipline
  • Reproduce first — confirm the bug exists before theorizing
  • One variable at a time — never shotgun multiple fixes simultaneously
  • Trace, don't guess — read logs, add instrumentation, inspect state
Investigation protocol
  • Start from error/symptom, work backward to root cause
  • Check recent changes first (git log, git diff)
  • Query memory for prior encounters with same error pattern
  • Use binary search (bisect) for regressions with long history
Prohibitions
  • Don't rewrite surrounding code while debugging
  • Don't add defensive checks to mask symptoms
  • Don't abandon a theory without disproving it
03
docs
documentation focus — docs.md

"Audience first. Show, don't tell. Accuracy over polish."

Core discipline
  • Audience first — API docs, tutorials, and architecture guides differ
  • Show, don't tell — a code example beats a paragraph of explanation
  • Accuracy over polish — verify claims against actual code
Writing standards
  • Lead with what the reader needs to do, not background history
  • Use concrete examples with realistic values, not foo/bar
  • One idea per sentence; consistent terminology throughout
Prohibitions
  • Do not document implementation details that will change
  • Do not modify code while in docs mode — docs files only
  • No filler padding ("In this section we will discuss...")
04
review
code review mode — review.md

"Read everything first. Assume intent. Severity matters."

Core discipline
  • Read everything first — all files in diff before commenting
  • Assume intent — if something looks wrong, ask why first
  • Severity matters — distinguish blockers from nits clearly
What to find
  • Logic errors, off-by-one, unhandled edge cases
  • Security: injection, auth bypass, data leaks, unvalidated input
  • Race conditions, shared state mutations, concurrency bugs
  • Breaking changes to public APIs or contracts
Output format
  • Group by severity: blockers, warnings, nits
  • Provide fix suggestion alongside every flagged issue
  • End with clear verdict: approve / request changes / needs discussion
05
skill
skill execution via /mode on <name> — skill/SKILL.md

"Skills are dormant mode overlays — activated by /mode on <name>, deactivated by /mode off."

Activation commands
  • /mode on <name> — activate named mode
  • /mode off — deactivate and delete _active_mode.md
  • /mode list — show available + currently active mode
File lifecycle
  • Copy modes/<name>.md to rules/_active_mode.md
  • Write mode name to modes/.active
  • Claude Code auto-loads any *.md in rules/
  • On deactivate, both files are deleted
Properties
  • Modes are purely additive — layer on CLAUDE.md + gates
  • Only one mode active at a time
  • Skills live in ~/.claude/skills/<name>/SKILL.md
  • SKILL.md is dormant until moved to skills/ slot
