02 — shared/plugin_registry.py 656 lines
Scans ~/.claude/plugins/ and all marketplace directories. Resolves plugin metadata (name, version, description, category, dependencies). Caches results to ramdisk (/dev/shm/claude-hooks/plugin_registry.json) with disk fallback. Design is fail-open — all public functions swallow exceptions so plugin failures never break gate enforcement. Also read-only — never modifies plugin files or settings.json.
  • scan_plugins() — enumerate available plugins with metadata
  • resolve_metadata() — infer category from name/description
  • get_enabled_plugins() — filter by settings.json enabledPlugins
  • Known categories: code-quality, testing, security, docs, infra
03 — shared/capability_registry.py 475 lines
Agent capability mapping for dynamic task routing and access control. Provides a structured mapping of agent capabilities to task requirements, enabling intelligent routing. The ACL subsystem enforces per-agent-type permissions via define_agent_acl() and check_agent_permission(). Uses fnmatch for pattern-based permission matching.
  • match_agent() — route task to best-fit agent type
  • recommend_model() — suggest opus/sonnet based on task complexity
  • get_agent_info() — return full capability descriptor
  • define_agent_acl() / check_agent_permission() — ACL enforcement
