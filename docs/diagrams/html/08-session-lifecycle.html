03 — Phase Details

SessionStart
  • boot_pkg/orchestrator.py — 22-step init
  • context.py — load session state, LIVE_STATE
  • maintenance.py — cleanup stale data
  • memory.py — connect to memory_server, write sideband
  • util.py — ramdisk mount, dir setup
  • Presents handoff: "Continue" or "New task"
UserPromptSubmit
  • user_prompt_capture.py — log prompt to session
  • statusline.py — inject warnings into context
  • State-based warnings: unlogged errors, pending fixes
  • Warnings rendered inline before Claude sees the prompt
  • Gate 19 (Hindsight) can trigger via UserPromptSubmit path
PreToolUse
  • enforcer_shim.py → daemon socket → gate_router
  • 17 active gates (01-07, 09-11, 13-19)
  • Gates 08 and 12 dormant
  • Tier 1 (01, 02, 03): fail-closed, exceptions block
  • All others: fail-open
  • Returns GateResult with severity + escalation
  • Exit 0 = allow, Exit 2 = block (PreToolUse only)
PostToolUse
  • tracker_pkg/orchestrator.py — 17-step pipeline
  • auto_remember.py — candidate extraction
  • errors.py — error detection + flagging
  • mentor.py + mentor_analytics.py — quality scoring
  • mentor_memory.py — persist mentor state
  • observations.py — compress and store
  • outcome_chains.py — link tool results to chains
  • verification.py — mark steps as verified
PreCompact
  • pre_compact.py — fires before context compaction
  • Snapshots current gate state to ramdisk
  • Flushes pending observation queue
  • Writes partial session summary to LIVE_STATE
SessionEnd + Stop
  • session_end.py — flush queues, update LIVE_STATE.json
  • Writes final session summary and "What's Next"
  • stop_cleanup.py — daemon teardown, socket close
  • tts_signal.py — optional audio notification
  • Ramdisk state preserved across sessions

04 — State Persistence

Ramdisk
  • Mount: /run/user/1000/claude-hooks/
  • state/ — per-session JSON files
  • audit/ — gate audit log rotation
  • Keyed by session_id
  • Sub-agent state isolated by session
LIVE_STATE.json
  • Written at SessionEnd by session_end.py
  • Read at SessionStart by boot orchestrator
  • Contains: current branch, project, "What's Next"
  • Session summary + handoff context
  • Gate counts and mentor scores
Audit + Config
  • audit_log.py — structured gate events
  • Rotated daily, stored on ramdisk
  • settings.json — 14 hook event registrations
  • config.json — 14 gate toggles + model_profile
  • ~/.claude.json — MCP server registrations
