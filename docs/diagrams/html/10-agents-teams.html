Delegation Rules
2-5
independent tasks → parallel sub-agents. All spawn simultaneously; memory MCP bridges results across instances.
2-5
dependent tasks → lead orchestrates sequentially. Memory MCP passes context from each step to the next.
5-7
medium scope → either pattern. Teams preferred for coherence — fewer context gaps between steps.
7+
large scope → agent teams mandatory. Structured coordination with lead + specialized member roles.
∞
cross-session → agents + LanceDB memory. search_knowledge bridges all prior context automatically across any number of sessions.
subagent_context.py — Context Injection Wiring
1. TRIGGER
SubagentStart hook fires every time the main session spawns a sub-agent via the Task tool. Reads agent_type from stdin JSON.
2. LIVE STATE
Loads ~/.claude/LIVE_STATE.json — project name, current feature, work summary, last session outcome and status.
3. SESSION STATE
Finds most recent hooks/state_*.json by mtime. Extracts files read, errors encountered, test status, bans, and active context.
4. INJECT
Tailored context block prepended to the sub-agent system prompt. Write-capable agents also receive EDIT_AGENT_RULES (read-before-edit, no destructive commands).
5. FAIL-OPEN
Always exit(0). If LIVE_STATE or session state cannot be read, falls back to FALLBACK_CONTEXT message — never crashes the agent spawn.
