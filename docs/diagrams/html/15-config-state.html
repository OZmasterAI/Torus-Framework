04 — LIVE_STATE.json
session_count
Monotonically incrementing session counter across all conversations.
project / feature
Current project name and active feature branch. Set at SessionStart.
framework_version
Torus Framework version string (e.g., v2.5.3). Checked against git tag.
what_was_done
Prose summary of work completed in the last session. Used for handoff context.
known_issues
Array of outstanding issues carried forward. Surfaces in SessionStart warning.
next_steps
Array of planned next actions. Shown in statusline and boot summary.

07 — Ramdisk
  • Mount: /run/user/1000/claude-hooks/
  • Subdirs: state/ and audit/
  • Speed: 544 MB/s vs 1.2 MB/s on disk
  • Init: setup_ramdisk.sh
  • Async disk mirror for persistence
08 — State Management
state.py
load_state() / save_state() — atomic fcntl.flock locking.
Per-session isolation keyed by session_id.
Schema v3, forward-compatible via state_migrator.py.
New fields added to default_state(), never removed.
Ramdisk path
State files live at /run/user/1000/claude-hooks/state/{session_id}.json.
Async mirror to ~/.claude/hooks/.state_backup/ on writes.

Config loading order: settings.json is loaded by Claude Code itself at startup. config.json is read by boot_pkg/orchestrator.py at SessionStart and hot-reloaded by config_change.py on writes. settings.local.json is merged over settings.json if present — safe for per-machine overrides without touching the tracked file.
