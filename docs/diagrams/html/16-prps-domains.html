03 — domains/ Structure
domains/framework/
Single registered domain. Contains behavior.md (behavioral overlay rules), mastery.md (framework expertise guidance), and profile.json (domain metadata).
shared/domain_registry.py
Scans domains/, registers entries, and connects domain selection to search_knowledge routing. Domain-specific rules are injected into the active agent's context.
  • behavior.md — hooks constraints, edit discipline, memory system overview
  • mastery.md — framework expertise: testing, patterns, upgrade paths
  • profile.json — domain identifier, version, match rules
04 — Domain Registration & Routing
How domains connect to search_knowledge: When a session starts, domain_registry.py scans the domains/ directory and loads matching profiles. The active domain's behavior.md is injected into Claude's context. Domain rules gate which knowledge topics are surfaced — the framework domain biases memory queries toward architecture, testing, and hook patterns. Dormant domains live in dormant/rules/domains.md.dormant and are excluded from routing until re-activated.

Note
Gate 9 auto-defer: After each PRP task completes, Gate 9 captures feedback signals and defers low-priority observations to avoid context bloat. Memory continuity between tasks is handled by memory-prefetch.py — it pre-queries the FTS5 index before each executor spawn so the fresh Claude instance has relevant context without reading the full history.
