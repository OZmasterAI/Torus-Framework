$ integration details
01 / telegram-bot
telegram-bot
bidirectional Claude Code mirror via Telegram
storage
  • msg_log.db — FTS5 virtual table + tg_meta
  • Schema: tg_fts(text, date, sender, chat_id, msg_id)
  • Multi-chat support; idx on chat_id + logged_at DESC
hooks
  • on_session_end.py — called by Stop hook
  • Reads LIVE_STATE.json → posts session summary to Telegram Bot API
  • Also logs message to msg_log.db FTS5
  • on_session_start.py — searches FTS5 for context relevant to new session
components
  • bot.py — Telegram Bot API handler
  • claude_runner.py — runs Claude from Telegram message
  • tmux_runner.py — sends text to active tmux pane
  • search.py — FTS5 query interface
  • sessions.py — session state tracking
memory cascade
  • L3 tier — Telegram messages cascade into LanceDB
  • MCP tool: telegram_search
Bot API SQLite FTS5 Stop hook L3 memory
02 / terminal-history
terminal-history
FTS5 index of all Claude Code session transcripts
storage
  • terminal_history.db — FTS5 SQLite, 19.8 MB
  • Indexes all JSONL session files in ~/.claude/projects/
  • Supports inherit+derive tagging from conversation content
  • FTS5 mirror copy at ~/data/memory/fts5_index.db
indexer
  • indexer.py — JSONL session parser + bulk indexer
  • Parses user/assistant turns; derives tags via regex rules
  • Derive rules: error/traceback → type:error, fix/resolved → type:fix
  • Marks indexed sessions; supports --retag to re-derive
search
  • search.py — CLI: python3 search.py "query"
  • Human-readable or --json machine output
  • Returns: text, timestamp, role, session_id, BM25 score, tags
hooks
  • on_session_end.py — indexes completed session
memory cascade
  • L2 tier — session content in LanceDB observations
  • MCP tool: terminal_history_search
JSONL parser FTS5 BM25 19.8 MB session_end hook L2 memory
03 / voice-web
voice-web
WebSocket bridge for iPad speech input + TTS output
architecture
  • server.py — Starlette/uvicorn WebSocket + HTTPS server
  • Routes: WebSocket, static files, TLS via self-signed cert
  • Fire-and-forget: sends transcribed text to tmux pane
  • Reads TTS signal file written by tts_signal.py (Stop hook)
TTS engines
  • Piper TTS — local neural voices, piper-voices/*.onnx
  • edge-tts — Microsoft Edge TTS cloud fallback
  • Lazy-loads Piper models on first request; cached per name
  • Returns WAV audio bytes over WebSocket
input flow
  • iPad microphone → WebSocket → server.py
  • server.py → tmux send-keys → Claude Code input
  • Claude response → tts_signal.py writes signal file
  • server.py reads signal → synthesizes speech → iPad
hook trigger
  • tts_signal.py fires on Notification event
  • Writes response text to signal file for server.py to consume
WebSocket Starlette Piper TTS edge-tts Notification hook Stop hook (TTS)
