02 — Gate Enforcement per Step

1 memory check
search_knowledge
  • Gate 04 memory_first
  • Reads .memory_last_queried sideband file
  • Blocks edits if memory not queried in 5-min window
  • Written atomically by boot.py at session start
  • Updated on every search_knowledge call
2 plan
explore + verify
  • Gate 14 confidence_check
  • Progressive warning on unverified file edits
  • Escalates when editing files not yet Read this session
  • Never write code in plan mode — ExitPlanMode first
  • Max 1 ExitPlanMode per turn
3 tests first
write or run tests
  • Gate 03 test_before_deploy
  • Must run tests before any deploy command
  • Checks for test runner (package.json, Makefile, etc.)
  • Enforced on Bash tool calls touching deploy/run scripts
4 build
read files first
  • Gate 01 read_before_edit (Tier 1, fail-closed)
  • Blocks Edit/Write unless file was Read this session
  • Gate 16 code_quality
  • Catches common quality issues in edits (large deletions, etc.)
5 prove it
show test evidence
  • Gate 05 proof_before_fixed
  • Never claim "fixed" without showing test output
  • Blocks final "done" claims if no verification seen
  • Counts unverified files per session
  • Blocks at threshold, not graduated warning
6 track
save to memory
  • Gate 06 save_fix
  • Auto-saves verified fixes to memory
  • Gate 15 causal_chain
  • Enforces query_fix_history before record_attempt
  • Tracks strategy_id → chain_id → outcome links
7 ship
commit or deploy
  • Gate 02 no_destroy (Tier 1)
  • Blocks rm -rf, reset --hard, push --force to main
  • Requires explicit user confirmation for destructive commands
  • Gate 07 critical_file_guard
  • Protects enforcer.py, memory_server.py, auth files
  • Extra confirmation required for critical path changes
