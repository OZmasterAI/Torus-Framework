Hook Pipeline

torus-framework — all 14 hook events — 2026-03-06

Every Claude action passes through a layered hook system. settings.json maps each event type to one or more handler scripts. Handlers receive JSON on stdin, run logic, and exit 0 (allow) or 2 (block — PreToolUse only). The diagram below shows the dispatch topology; the cards expand each event type with full flow detail.

Event Cards

01 · PreToolUse
can block
Edit | Write | NotebookEdit | Bash | Task | Skill
WebFetch | WebSearch | LSP | EnterWorktree
enforcer_shim.py → UDS socket → enforcer_daemon.py → gate_router.py → 17 gates → GateResult
Exit 0 allow  ·  Exit 2 block.
Fallback: inline enforcer.py if daemon down (~134ms vs ~43ms daemon).
02 · PostToolUse
informational
* (all tools)
event_logger.py — logs all tool results to audit trail.

auto_format.py (Edit|Write) — ruff/black, 3s timeout.

tracker.py (Edit|Write) — 17-step pipeline via tracker_pkg/; mentor system for quality signals.

auto_commit.py (Edit|Write) — stages files for deferred commit.
03 · SessionStart
lifecycle
* (all sessions)
boot.py → boot_pkg/orchestrator.py — 22-step pipeline:
ramdisk init → audit rotation → LIVE_STATE load → UDS check → memory inject → gate auto-tune → context inject → state reset
04 · SessionEnd
lifecycle
* (all sessions)
session_end.py:
flush queues · update LIVE_STATE.json · increment session_count · backup LanceDB tables.
05 · UserPromptSubmit
capture
* (all prompts)
user_prompt_capture.py — captures prompt text to observation queues.

statusline.py — injects warnings when: uncommitted changes present; memory not queried this turn.
06 · PermissionRequest
passive
* (all permission checks)
auto_approve.py — auto-approves known-benign tools to reduce interactive interrupts.
07 · SubagentStart
context inject
* (all subagent launches)
subagent_context.py — injects LIVE_STATE.json and per-session state into the subagent's context so sub-agents share project awareness automatically.
08 · SubagentStop
logging
* (all subagent completions)
event_logger.py — records subagent completion with duration and result status to the audit trail.
09 · PreCompact
lifecycle
* (before context compression)
pre_compact.py — snapshots current gate state, queue depths, and session metrics before context window compression so continuity is preserved across the compaction boundary.
10 · Stop
lifecycle
* (Claude response complete)
stop_cleanup.py — flushes I/O buffers, shuts down background daemons cleanly.

tts_signal.py — emits audio signal indicating response is ready.
11 · PostToolUseFailure
error recovery
* (any tool error)
failure_recovery.py — records failure context, triggers retry hints, escalates to state if recurring.

event_logger.py — captures error trace to audit log.
12 · Notification
signal
* (system notifications)
tts_signal.py — converts notification events to audio signals for ambient awareness without screen focus.
13 · ConfigChange
hot-reload
user_settings | project_settings
config_change.py — hot-reloads config.json changes into the running session without restart. Gate thresholds, model budgets, and bypass lists update immediately.
Exit code contract: Exit 0 = allow. Exit 2 = block (PreToolUse only — stderr shown to Claude). Exit 1 = non-blocking error (logged, proceeds). Only PreToolUse handlers can halt an action. All other event types are informational.
