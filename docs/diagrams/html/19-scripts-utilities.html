torus-framework — orchestration, maintenance, and boot utilities

01 — ~/.claude/scripts/ — Orchestration

Script Lines When Description
torus-loop.sh 261 manual Sequential fresh-context task executor. Spawns a fresh claude -p process per task from a PRP's tasks.json. Each spawn gets peak reasoning quality — no context degradation. Memory MCP bridges knowledge between instances.
torus-wave.py 477 manual Parallel wave orchestrator. Groups independent tasks into waves (using file-overlap guards to prevent conflicts), spawns parallel claude -p processes, serializes validation, and loops until all tasks pass or max iterations reached.
torus-prompt.md — injected Prompt template injected into each executor invocation by loop/wave scripts. Provides task context, memory prefetch output, and framework behavioral constraints.
memory-prefetch.py 98 per-task Pre-fetches relevant memories for a task executor prompt. Queries the FTS5 index (read-only, safe for concurrent access) using task name and file list. Returns top 5 relevant memories. Exit code always 0 (fail-open).
prp-phase-verify.py 208 on-demand Post-phase verification for /prp verify-phase. Runs task_manager.py verify-phase, analyzes failures by level, and optionally creates fix tasks. Exit 0 = all passed, exit 1 = failures found.
prp-plan-verify.py 135 on-demand Plan verification loop for /prp plan-phase. Runs task_manager.py plan-check against requirements, analyzes gaps, outputs revision report. Used by the /prp skill to decide whether to revise the plan.
cleanup-x-sessions.sh 150 manual Removes stale X session lock files and sockets. Iterates /tmp/.X*-lock files, checks if the owning PID is alive. Dead PIDs: remove lock + socket. Supports X_CLEANUP_DRY_RUN=true dry run mode.

02 — ~/.claude/hooks/scripts/ — Maintenance

Script Lines When Description
audit_rotation.py 160 cron/daily Compresses .jsonl audit logs older than 7 days to .jsonl.gz. Deletes .jsonl.gz files older than 30 days. Reports disk usage before and after. Manages the framework's full audit trail.
backup_embeddings.py 109 manual Exports all 5 LanceDB tables (documents, metadata, embeddings, IDs) to JSON at ~/data/memory/backup_lance_YYYYMMDD.json. Full backup of the knowledge, observations, fix_outcomes, quarantine, and web_pages tables.
create_lance_indexes.py 71 manual Creates IVF_PQ vector indexes on LanceDB tables for fast approximate nearest-neighbor search. Useful when tables grow beyond ~10K rows. No-op if indexes already exist.
fix_lance_indexes.py 38 manual Drops IVF_PQ indexes that hurt relevance on small tables. For datasets under ~10K rows, brute-force cosine scan is faster and more accurate. LanceDB falls back to flat scan when no index exists.
flush_audit.py 72 cron Flushes old ramdisk audit logs to disk backup (compressed). Keeps last KEEP_DAYS days on ramdisk, compresses older files to ~/.claude/hooks/audit/ as .gz, then removes originals from ramdisk.
memory_compactor.py 315 manual Smart deduplication for LanceDB knowledge table. Scans for similar memories (cosine similarity above threshold), clusters them, merges metadata into survivors, quarantines duplicates. Defaults to dry-run mode.
stale_memory_archiver.py 247 manual Identifies and demotes old low-value memories. Scans knowledge table for entries older than 90 days with low-value heuristics (short text, no tags, low tier). Demoted memories move to tier 3 so they rank lower in search.

Design Notes
Fail-open by design. All maintenance scripts (hooks/scripts/) default to dry-run or fail gracefully — they never crash gate enforcement. The memory-prefetch.py always exits 0 so a failed memory lookup never blocks task execution. Fresh context is the key insight behind both torus-loop and torus-wave: rather than letting one long session degrade, each task gets a virgin Claude instance that queries memory to re-establish context just-in-time.
