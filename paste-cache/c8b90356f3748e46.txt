
  2. The ChromaDB test fix

  The fix is wrapping the regression test block in the existing guard. Right now it looks roughly like:

  # ~line 1795 (module level, runs during import)
  _client = chromadb.PersistentClient(path=MEMORY_DIR)
  _obs_col = _client.get_or_create_collection(name="observations", ...)
  _know_col = _client.get_or_create_collection(name="knowledge", ...)

  _sample_obs = _obs_col.get(limit=10, include=["metadatas"])  # <-- BOOM

  The fix would be:

  if not MEMORY_SERVER_RUNNING:
      _client = chromadb.PersistentClient(path=MEMORY_DIR)
      _obs_col = _client.get_or_create_collection(...)
      _know_col = _client.get_or_create_collection(...)
      _sample_obs = _obs_col.get(limit=10, include=["metadatas"])
      # ... rest of regression tests
  else:
      skip("Regression: session_time type check", "memory server running")
      skip("Regression: existing observations type check", "memory server running")

  Would it break anything? No. The tests already can't run — they crash during collection. This just makes the crash into a graceful skip.

  Would we lose anything? The regression tests still run when the MCP server is not running (e.g., in CI, or if you stop the server first). You only lose them during live sessions where the server holds the DB lock — which is exactly when they can't
  work anyway. Going from "crash that kills all 1043 tests" to "skip 2 tests" is a net gain.
